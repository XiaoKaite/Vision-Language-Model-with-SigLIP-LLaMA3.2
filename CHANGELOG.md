# 原项目更新说明

## 新增文件

| 文件 | 功能 |
|---|---|
| extract_frames.py | 视频抽帧，每秒截一张图保存到本地 |
| batch_inference.py | 批量推理，遍历所有帧图片生成描述并输出 JSON |
| run.py | 一键串联脚本，自动完成"视频到抽帧到推理" |

## 与原项目 inference.py 的对比

### 模型加载效率

原项目每处理一张图片就重新加载一次模型，100 张图要加载 100 遍，严重浪费时间和显存。

改良后模型只加载一次，后续反复调用推理方法，100 张图只加载一次。以加载模型耗时约 30 秒估算，处理 100 帧时原项目光加载就要 50 分钟，改良后只需 30 秒。

### 路径配置

原项目所有路径写死在代码里（D:\prj_wsl\...），换一台电脑就要手动改代码。

改良后路径通过命令行参数传入，换设备不用改一行代码。

### 输入输出模式

原项目一次只能处理一张图，结果打印到屏幕，不做保存。

改良后自动扫描目录下所有图片文件，按时间顺序排列，逐帧推理并汇总为结构化 JSON 文件，每条记录包含秒数、文件名、描述文本，方便后续查阅和分析。

### 推理逻辑

视觉编码、投影、生成的核心流程与原项目完全一致，未做修改。改良仅涉及工程层面的封装和批处理。

## 效果对比（以 60 秒视频为例）

| 对比项 | 原项目 | 改良后 |
|---|---|---|
| 模型加载次数 | 60 次 | 1 次 |
| 改代码换路径 | 需要 | 不需要 |
| 输出格式 | 屏幕打印 | JSON 文件 |
| 操作步骤 | 手动抽帧 + 逐个推理 | 一条命令完成 |
| 总耗时估算 | 约 30 分钟加载 + 实际推理时间 | 约 30 秒加载 + 实际推理时间 |
